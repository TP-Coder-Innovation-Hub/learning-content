# AI Research

## What

AI can accelerate research by summarizing, synthesizing, and connecting information. But it is a starting point, not a final answer. Every AI-generated claim needs verification.

## The Workflow

### Step 1: Frame the Question

Be specific. "Tell me about databases" gives you a textbook summary. "Compare PostgreSQL and DynamoDB for an e-commerce order management system with 1000 writes/second" gives you a useful analysis.

### Step 2: Get the Overview

Ask AI for a high-level summary of the topic. This gives you a mental map and vocabulary to search further.

```
Give me an overview of distributed tracing. Focus on:
- What problem it solves
- Core concepts (trace, span, context propagation)
- How it differs from logging and metrics
- Main tools in the ecosystem
```

### Step 3: Go Deep on Specifics

Follow up with targeted questions.

```
How does context propagation work in a microservices architecture 
using HTTP? Show me the headers involved and how they're passed 
from service to service.
```

### Step 4: Verify

This is the critical step. AI generates plausible-sounding information. Some of it is wrong.

Verify:
- **Facts** — Check dates, version numbers, names against primary sources
- **Code examples** — Run them. Do they actually work?
- **Claims** — "Best practice" according to AI might be outdated or contested
- **Technical details** — API signatures, configuration values, syntax

Sources for verification:
- Official documentation
- Source code and GitHub repositories
- Published papers and conference talks
- Experienced practitioners

### Step 5: Synthesize

Combine AI-generated research with verified sources into your own understanding. The output should be yours, informed by AI, not generated by AI.

## What AI is Good For

- **Getting started fast** — Overcome the blank page problem
- **Connecting concepts** — "How does X relate to Y?"
- **Summarizing long documents** — Paste a research paper and ask for key findings
- **Generating search queries** — "What terms should I use to research event sourcing?"
- **Comparing options** — "What are the trade-offs between Kafka and RabbitMQ?"

## What AI is Bad For

- **Being a primary source** — It is trained on data, not experience. It doesn't know what works in your specific context.
- **Up-to-date information** — Training data has a cutoff. Recent developments may not be reflected.
- **Nuanced opinions** — It tends to present balanced "on the other hand" views that don't help you make a decision.
- **Domain-specific expertise** — It knows common knowledge well. Niche topics less so.

## Practical Tips

- Ask "What sources should I read to learn more about this?" Then read those sources.
- Use AI to generate a list of questions to investigate. Then investigate them yourself.
- Cross-reference multiple AI responses. If two outputs disagree, neither is trustworthy without verification.
- Keep notes of what you verified and what you didn't. If you use unverified information, label it.

## Common Mistakes

- Treating AI output as research. It is a starting point, not a source.
- Not verifying code examples. They look correct but may use deprecated APIs, wrong syntax, or hallucinated functions.
- Asking one big question instead of several focused ones. Small, specific questions get better answers.
- Stopping at the AI response. The real research starts after AI gives you the overview.
